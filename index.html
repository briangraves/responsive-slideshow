What is Responsive Design?

  • Provides an optimal viewing experience for websites across a range of devices, large and small
  • The user interface adapts to the size of the screen via a fluid layout, flexible images and CSS Media Queries
  • The experience is optimized for device capabilities (touch)
  • The first ever website was responsive: http://www.w3.org/History/19921103-hypertext/hypertext/WWW/TheProject.html"

Why Would You Use Responsive Design?

  • One Content Base
  • One Code Base
  • Lines Between Device Categories are Becoming More Blurred
  • Feature Detection Allows for Better Means of Determining Context
  • Future Friendly - Sites Will Be Built to Naturally Adapt to New Device Categories and Screen Sizes

Build your responsive site responsibly

  • Content
    • Responsive web designs can often consume lots of bandwidth when resources and assets aren't handled correctly.
    • Don't hide content for the sake of hiding content.
  • Design with touch in mind.
    • Tap area sizes
    • Touch interactions
    • Swiping
  • Current technology road blocks such as existing content, CMS etc...
  • Will not happen over night, do plenty of research

What can I do to get you into a responsive website today?

  1. From Scratch. The most efficient and preferred way to build a responsive website is from the ground up. This option takes advantage of using a proper project workflow and making sure the user gets the best experience.
  2. Retrofit. Can provide a better user experience for smaller devices on an existing website.
  3. Start with an "m dot". Leave the existing desktop website where it is. Create an m.* mobile site and make it responsive to eventually fade it into all devices. People magazine took this approach. Mobile | Desktop
